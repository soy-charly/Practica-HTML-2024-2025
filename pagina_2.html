Comandos Básicos de GitHub

Comandos Esenciales de Git y GitHub

Clonar un Repositorio

Para empezar a trabajar con GitHub, primero debes clonar un repositorio.
Esto te permite descargar una copia local del proyecto en tu máquina.
El comando git clone seguido de la URL del repositorio clona el contenido.

Ejemplo: git clone https://github.com/usuario/proyecto.git

Realizar un Commit

Después de realizar cambios en tu proyecto, puedes guardar estos cambios
localmente con el comando git commit.

Primero debes agregar los archivos al área de preparación con git add .
y luego realizar el commit con un mensaje que describa los cambios.

Ejemplo: git commit -m "Descripción de los cambios"

Subir Cambios a GitHub

Una vez realizados los commits, puedes subir los cambios a GitHub con
el comando git push. Este comando envía los cambios locales
al repositorio remoto en GitHub.

Ejemplo: git push origin main subirá los cambios a la rama principal.

Crear una Rama

Al trabajar en GitHub, es buena práctica crear ramas para trabajar en nuevas características
sin afectar la rama principal. Puedes crear una nueva rama con el comando git branch
y cambiar a ella con git checkout.

Ejemplo: git branch nueva-rama seguido de git checkout nueva-rama

Hacer un Pull Request

Una vez que hayas terminado tu trabajo en una rama, puedes hacer un
Pull Request en GitHub. Esto permite que los colaboradores
revisen tu trabajo antes de fusionarlo con la rama principal.

Para hacer un Pull Request, ve al repositorio en GitHub, selecciona tu rama
y haz clic en el botón "New Pull Request".

Fusionar Ramas (Merge)

Cuando se apruebe tu Pull Request, puedes fusionar tu rama con la principal
utilizando el comando git merge. Esto aplica todos los cambios
de tu rama a la rama principal.

Ejemplo: git merge nueva-rama
